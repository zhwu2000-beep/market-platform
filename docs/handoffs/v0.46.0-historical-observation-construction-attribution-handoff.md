## Observation Call Graph

Production entry remains `HistoricalReplayService.run()` in `src/market_platform/replay/service.py`.

For each replay position:

1. `_copy_replay_prefix(normalized, position)` creates `normalized.iloc[: position + 1].copy(deep=True)`.
2. The current full-frame row timestamp becomes `as_of` through replay `_to_datetime(...)`.
3. `signal_snapshot = signal_snapshots[position]` reads full-frame precomputed signal state.
4. `structure_snapshot = structure_snapshots[position]` reads full-frame precomputed structure state on the default path. Injected/custom structure services still use `self._price_structure_service.analyze(prefix, as_of=as_of)`.
5. `build_historical_market_observation(prefix, symbol, interval, as_of, provider, signal_snapshot, structure_snapshot)` remains the only production historical observation builder entry.
6. The builder normalizes `as_of`, calls `_normalize_price_prefix(prices, as_of)`, normalizes symbol/provider/interval metadata, validates snapshot types, validates prefix symbol/provider, and constructs:
   - `ObservationIdentity`
   - `ObservationProvenance`
   - `PriceFacts`
   - `MarketObservation`
7. `ObservationProvenance` calls `_historical_observation_fingerprint(...)`, which builds a payload containing `symbol`, `provider`, `interval`, `as_of`, and every normalized prefix row, canonicalizes it with sorted JSON keys, and hashes UTF-8 bytes with SHA-256.
8. `build_market_observation(...)` adapts signal and structure snapshots through `build_signal_facts(...)` and `build_structure_facts(...)`, then constructs `MarketObservation`.
9. Downstream, `state_model.evaluate(observation)` consumes the full observation, and `StrategyRunner.evaluate(..., observation=observation, state=state)` consumes the same observation. `HistoricalReplayStep` stores the observation fingerprint, state, and strategy result.

No final observation serialization, replay result serialization, structure reporting, or signal reporting happens inside the historical observation builder. The builder only performs canonical JSON serialization for the input fingerprint.

## Attribution Method

The benchmark keeps two tracks:

- Production timing: direct uninstrumented `HistoricalReplayService.run()`.
- Instrumented attribution: call-through wrappers patched into the actual production bindings while still calling real `HistoricalReplayService.run()`.

Instrumentation patches are restored through `ExitStack` on normal exit and exceptions. The benchmark still asserts exact `HistoricalReplayResult.to_dict()` parity between production and instrumented runs.

Observation internal segments are reported as exclusive timings where child timings exist:

- `observation_prefix_preparation`
- `observation_metadata`
- `observation_identity`
- `observation_price_facts`
- `observation_fingerprint_rows`
- `observation_fingerprint_canonical_json`
- `observation_fingerprint_hash`
- `observation_provenance` exclusive of fingerprint child timing
- `observation_signal_facts`
- `observation_structure_facts`
- `observation_model_construction` exclusive of signal/structure fact child timing
- `observation_residual`, computed as observation construction total minus measured exclusive child segments

Structure internals remain nested under `structure_precompute`. Final replay result serialization remains measured after `HistoricalReplayService.run()` returns.

## 500-Bar Segment Attribution

500-bar observation construction median: `4223.604 ms`.

| Segment | Median | Observation share |
| --- | ---: | ---: |
| Prefix preparation | 2193.707 ms | 51.9% |
| Fingerprint row extraction | 1380.667 ms | 32.7% |
| Canonical JSON | 204.710 ms | 4.8% |
| Hash | 19.256 ms | 0.5% |
| Price facts | 103.462 ms | 2.4% |
| Identity | 98.666 ms | 2.3% |
| Provenance exclusive | 33.689 ms | 0.8% |
| Signal facts | 11.171 ms | 0.3% |
| Structure facts | 8.860 ms | 0.2% |
| Model construction | 9.812 ms | 0.2% |
| Residual | 161.731 ms | 3.8% |

Measured conclusion: the largest cost is repeated prefix preparation, not `PriceFacts` construction and not SHA-256 hashing itself. Fingerprint row extraction is the adjacent second-largest prefix-dependent cost.
