## Structure Precompute Design

The new internal helper is `precompute_price_structure_snapshots(...)` in `src/market_platform/structure/precompute.py`.

It performs once per full normalized frame:

- Structure frame validation and normalization.
- UTC timestamp normalization.
- Stable timestamp sorting.
- Duplicate timestamp validation.
- Numeric high/low/close normalization.
- Default swing-high candidate detection.
- Default swing-low candidate detection.
- Deterministic candidate sorting.
- True-range and rolling ATR series construction.
- Position/timestamp mapping through full-frame row order.

For each full-frame position, it still performs exact point-in-time work:

- Candidate visibility filtering by `confirmed_at <= snapshot_as_of`.
- Snapshot construction using that position's ATR.
- Clustering and zone construction from currently visible candidates.
- Touch observation against `prepared.iloc[: position + 1]`.

Replay integration:

- Default replay precomputes `structure_snapshots` before the replay loop.
- Replay step `position` reads `structure_snapshots[position]`.
- The replay loop still builds a deep-copy prefix for the observation builder.
- Public `PriceStructureService.analyze()` remains unchanged.
- The structure package does not depend on replay.
- The batch helper is not exported from `market_platform.structure.__all__`.

## Point-in-Time Guarantees

The default pivot model has both `occurred_at` and `confirmed_at`.

Point-in-time rules:

- A pivot first becomes legally visible at its confirmation position.
- A pivot is invisible before `confirmed_at`.
- Future bars in a full frame must not change earlier snapshots.
- Tail pivots without a complete right window must not appear early.
- Replay start/end only filters full-frame positions.
- Subwindows do not restart structure warm-up.
- The exact parity oracle is the public prefix call:

```python
PriceStructureService().analyze(
    normalized.iloc[: i + 1],
    as_of=normalized.iloc[i]["timestamp"],
)
```

## Benchmark Methodology

The benchmark method is unchanged from v0.43.0 except for new structure-precompute and internal attribution fields:

- Deterministic generator.
- Fixed seed: `43`.
- Max-bars frame generated once.
- 100/300/500 scenarios use prefixes from the same max frame.
- Two default strategies.
- DataFrame construction outside timed replay runs.
- Warm-up runs separated from measured runs.
- `gc.collect()` before each warm-up and measured run.
- Uninstrumented production timing and instrumented attribution are separate tracks.
- Serialization is measured after `HistoricalReplayService.run()` returns.
- Machine load can cause visible run-to-run wall-clock variance.
- Single benchmark runs are not cross-machine performance promises.

The two v0.44.0 runs used the same production code path. The review run was slower across all scenarios and is most likely local machine load or sample noise, not a code-path regression.

Correct interpretation:

- The first run showed limited gains: 1.39x, 1.13x, and 1.06x.
- The review run should not be used alone to claim a stable speedup or regression.
- v0.44.0 is primarily a correctness-preserving infrastructure optimization and attribution foundation, not a major performance breakthrough.

## v0.45.0 Main Design Question

The next thread should first answer:

Can the system efficiently compute each point-in-time zone's prefix touch observation without scanning the full prefix for every position and every zone?

Investigate:

- Whether zone bounds change across positions.
- Whether ATR changes alter zone geometry for the same candidate set.
- Whether fixed zone bounds can use vectorized high/low membership.
- Whether touch prefix state can be computed once from a zone's first appearance through all later positions.
- Whether zone identity and geometry need to be separated.
- How to preserve consecutive-inside suppression.
- How to preserve first and last touch timestamps.
- How to avoid future data.
- Whether optimization should only cover the default observer.
- Whether per-prefix fallback is still required.
