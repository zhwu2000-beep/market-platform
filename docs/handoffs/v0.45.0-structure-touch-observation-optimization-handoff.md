## Touch Semantics Preserved

A candle touches a zone when its high-low range intersects inclusive zone bounds:

```text
low <= zone.upper_bound and high >= zone.lower_bound
```

Preserved behavior:

- Boundary equality counts as a touch.
- Only high and low participate.
- The first row counts if it is inside the zone.
- Consecutive inside candles count as one independent touch.
- Leaving and re-entering creates a new touch.
- `first_observed_at` and `last_observed_at` are entry-candle timestamps.
- `last_observed_at` is not the last candle of a still-open inside run.
- Count zero returns `None` for both timestamps.
- Touch scanning semantics cover the whole current prefix, not only bars after `zone.available_at`.

## Design

`precompute_price_structure_snapshots(...)` now creates a local in-call dictionary:

```python
dict[tuple[float, float], _TouchObservationSeries]
```

The exact touch key is:

```python
(zone.lower_bound, zone.upper_bound)
```

The key intentionally excludes midpoint, candidates, source methods, available_at, support/resistance classification, current price, and prior observation results. There is no rounding, quantization, approximate equality, persistence, module global state, or service-level cache.

For a fixed exact bounds key, `_build_touch_observation_series(...)` computes once over the full normalized frame:

```text
inside[i] = low[i] <= upper_bound and high[i] >= lower_bound
entry[0] = inside[0]
entry[i] = inside[i] and not inside[i - 1]
counts = cumulative_sum(entry)
last_entry_indices = prefix latest entry index
first_entry_index = first entry index or None
```

Stored state contains:

- `counts`: read-only NumPy int64 cumulative counts
- `last_entry_indices`: read-only NumPy int64 latest prefix entry indices
- `first_entry_index`: Python int or None
- `array_bytes`: retained array payload bytes for benchmark attribution

Temporary arrays (`inside`, `previous_inside`, `entries`, `entry_indices`) are build-time intermediates and are not retained in `_TouchObservationSeries`.

Every observed zone still gets a fresh `ObservedPriceZone` with the current `PriceZone` model. Only the touch observation state is reused.

## Point-in-Time Guarantees

For a zone `z` at full-frame position `i`, optimized touch state must remain equivalent to:

```python
observe_price_zone(normalized.iloc[: i + 1], z)
```

`observation_at(position)` reads `counts[position]` first. If the prefix count is zero, it returns `PriceZoneObservation(0, None, None)` even if the first touch occurs later in the full frame. It also validates `position` and raises `IndexError("position out of range")` for negative or out-of-range private lookups.

Future bars may be present in the full-frame series, but they cannot change earlier prefix counts, first timestamps, or last timestamps.

## Historical-Before-Availability Semantics

The optimization intentionally does not use `zone.available_at` as a touch-state start point. A zone first appearing at a later position can immediately report historical touches from earlier prefix rows if its bounds intersected those candles.

Tests cover a late-confirmed zone whose first snapshot includes historical touch count and first/last timestamps before `available_at`, matching the public whole-prefix observer.
