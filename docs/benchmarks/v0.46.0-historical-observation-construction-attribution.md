## Observation Call Graph

Production entry remains `HistoricalReplayService.run()` in `src/market_platform/replay/service.py`.

For each replay position selected from the full normalized frame:

1. `prefix = _copy_replay_prefix(normalized, position)` passes `normalized.iloc[: position + 1].copy(deep=True)` into observation construction.
2. `as_of = _to_datetime(normalized.iloc[position]["timestamp"])` reads the current full-frame row timestamp.
3. `signal_snapshot = signal_snapshots[position]` reads the precomputed signal snapshot.
4. `structure_snapshot = structure_snapshots[position]` reads the precomputed structure snapshot on the default path; injected/custom structure services still call `self._price_structure_service.analyze(prefix, as_of=as_of)`.
5. `build_historical_market_observation(prefix, symbol, interval, as_of, provider, signal_snapshot, structure_snapshot)` is the only production observation builder entry.
6. `build_historical_market_observation()` normalizes `as_of`, calls `_normalize_price_prefix(prices, as_of)`, normalizes symbol/provider/interval metadata, validates snapshot types, validates prefix symbol/provider, then calls private helpers:
   - `_build_observation_identity(symbol, interval, as_of, prices)` returns `ObservationIdentity`.
   - `_build_observation_provenance(prices, symbol, interval, as_of, provider)` returns `ObservationProvenance` and calls `_historical_observation_fingerprint()`.
   - `_build_price_facts(prices)` returns `PriceFacts` from the latest close and latest timestamp only.
   - `_construct_historical_observation(...)` calls `build_market_observation(...)`.
7. `_historical_observation_fingerprint()` calls:
   - `_historical_observation_fingerprint_payload(...)`
   - `_historical_observation_fingerprint_rows(prices)` over every row in the normalized prefix
   - `_canonicalize_historical_observation_fingerprint_payload(payload)` with `json.dumps(..., allow_nan=False, separators=(",", ":"), sort_keys=True)`
   - `_hash_historical_observation_fingerprint(canonical)` using SHA-256 over UTF-8 bytes.
8. `build_market_observation()` in `src/market_platform/observation/builder.py` calls:
   - `build_signal_facts(signal_snapshot)` -> `SignalFacts(symbol, as_of, signals)`; `SignalFacts` copies each `MarketSignal` and each signal parameter dict.
   - `build_structure_facts(structure_snapshot)` -> `StructureFacts(status, as_of, current_price, atr, confirmed_pivots, available_zones)`; structure tuples are normalized but structure models are not serialized.
   - `build_observation(...)` -> `MarketObservation(...)`.
9. Downstream, `state_model.evaluate(observation)` consumes `MarketObservation`, then `StrategyRunner.evaluate(..., observation=observation, state=state)` consumes the same observation. `HistoricalReplayStep` stores only `observation.provenance.input_fingerprint`, state, and strategy result.

No final `MarketObservation.to_dict()`, structure reporting, signal reporting, or replay-result serialization occurs inside the historical observation builder. Canonical JSON inside the builder exists only for the input fingerprint.

## Exact Semantics

Observation fields and sources:

- `identity.symbol`: normalized `symbol` argument, uppercase.
- `identity.interval`: normalized `interval` argument.
- `identity.as_of`: replay current row timestamp normalized to UTC.
- `identity.window_start`: first timestamp in normalized prefix.
- `identity.window_end`: last timestamp in normalized prefix.
- `provenance.provider`: single replay provider metadata passed from `HistoricalReplayService.run()`.
- `provenance.methodology`: constant `historical_replay_observation`.
- `provenance.methodology_version`: constant `1.0.0`.
- `provenance.parameters`: `{"interval": normalized_interval}`.
- `provenance.input_fingerprint`: SHA-256 over canonical JSON containing `symbol`, `provider`, `interval`, `as_of`, and every normalized prefix row's symbol/timestamp/OHLCV/provider.
- `price_facts.latest_price`: current/latest prefix row close only.
- `price_facts.observed_at`: current/latest prefix row timestamp only.
- `signal_facts`: copied from the precomputed `MarketSignalSnapshot` for the same full-frame position.
- `structure_facts`: copied from the precomputed `PriceStructureSnapshot` for the same full-frame position.

The observation does not include the full prefix as a model field. The fingerprint does include all prefix rows. The builder does not call DataFrame `to_dict()` or `to_json()`, but it manually iterates rows, creates dicts, canonicalizes with sorted JSON keys, and hashes the resulting UTF-8 bytes. UTC conversion happens during prefix normalization and again when timestamps are projected into identity, price facts, and fingerprint rows. Full-frame replay start/end semantics remain unchanged: signal and structure precompute run on the full frame, and start/end only filter replay positions.

## Instrumentation Design

The benchmark keeps two tracks:

- Production timing: direct uninstrumented `HistoricalReplayService.run()`.
- Instrumented attribution: still calls real `HistoricalReplayService.run()` and patches actual module bindings with call-through wrappers.

v0.46.0 adds private helper extraction only in `src/market_platform/observation/history.py`. Public `build_historical_market_observation()` remains the only production implementation. The benchmark patches these real helper bindings, plus `observation.builder.build_signal_facts` and `build_structure_facts`. `ExitStack` restores patches on normal exit and exceptions.

No replay loop is copied. The harness still asserts production and instrumented `HistoricalReplayResult.to_dict()` exact parity.

## Segment Definitions

Top-level replay segments are unchanged. New observation internal segments are exclusive unless noted by residual:

- `observation_prefix_preparation`: `_normalize_price_prefix()`; includes prefix DataFrame column copy, text normalization, timestamp UTC conversion, future-data check, numeric conversion, high/low validation, stable sort, duplicate timestamp check.
- `observation_metadata`: symbol/provider/interval normalization.
- `observation_identity`: `ObservationIdentity` construction and validation.
- `observation_price_facts`: latest-row close/timestamp extraction and `PriceFacts` construction.
- `observation_fingerprint_rows`: row-by-row fingerprint dict construction over the normalized prefix.
- `observation_fingerprint_canonical_json`: canonical JSON generation with sorted keys.
- `observation_fingerprint_hash`: SHA-256 digest over canonical UTF-8 bytes.
- `observation_provenance`: `ObservationProvenance` construction exclusive of fingerprint row/canonical/hash child time.
- `observation_signal_facts`: signal snapshot conversion into `SignalFacts`.
- `observation_structure_facts`: structure snapshot conversion into `StructureFacts`.
- `observation_model_construction`: `_construct_historical_observation()` exclusive of signal/structure fact child time.
- `observation_residual`: observation orchestration/model/unattributed residual after measured exclusive child segments.

Structure internals remain nested inside `structure_precompute`. Replay residual does not include structure internals or observation internals in its subtraction; it only subtracts top-level replay segments.

## Fingerprint And Canonicalization

Fingerprint algorithm:

- Inputs: normalized prefix rows, `symbol`, `provider`, `interval`, `as_of`.
- Excludes signal and structure snapshots.
- Iterates every prefix row with `DataFrame.itertuples(index=False)`.
- Converts timestamps to UTC datetimes and ISO strings.
- Normalizes OHLCV floats with `repr(float(value))`.
- Canonicalizes with `json.dumps(..., allow_nan=False, separators=(",", ":"), sort_keys=True)`.
- Hashes UTF-8 canonical JSON with SHA-256 and prefixes `sha256:`.

At 500 bars, fingerprint row extraction plus canonical JSON plus hash was `1604.634 ms`, about 38.0% of observation construction. Hashing itself was only `19.256 ms`; the expensive work is prefix row projection and JSON canonicalization, not SHA-256.

The fingerprint is prefix-dependent and cumulative: 500-bar replay canonicalized 125250 rows and 27866808 characters/bytes.

## Price Facts

`PriceFacts` uses only the latest normalized prefix row:

- latest close -> `latest_price`
- latest timestamp -> `observed_at`

There are no rolling aggregates, no prefix statistics, and no full-prefix price facts beyond validation and fingerprint input preparation. The measured 500-bar price-facts segment was `103.462 ms`, much smaller than prefix preparation and fingerprint rows.

## Signal And Structure Conversion

Signal conversion receives a complete `MarketSignalSnapshot`; each observation copies 5 `MarketSignal` values, including parameter dicts. At 500 bars this converted 2500 signal facts and cost `11.171 ms`.

Structure conversion receives a complete `PriceStructureSnapshot`; it normalizes existing candidate and observed-zone tuples into `StructureFacts` without snapshot reporting or serialization. At 500 bars this covered 11903 cumulative candidates, 8375 cumulative observed zones, and 55083 touch-count payload, costing `8.860 ms`.

Observation construction is therefore not primarily structure snapshot conversion, signal conversion, or model validation/construction.

## Measurement Limitations

- Observation internal wrappers add measurable overhead, especially at 100 bars. Use production medians as authoritative wall-clock and internal medians as attribution evidence.
- Top-level production and instrumented samples are independent; shares can exceed production total on noisy runs, as seen in 100-bar observation share.
- Segment timings are exclusive for observation internals, except residual which is computed by subtraction from measured observation total.
- No performance threshold tests were added.

## Decision

Decision: **GO: Observation Prefix Preparation Optimization**.

Rationale:

- Largest 500-bar segment: `_normalize_price_prefix()` at `2193.707 ms`, 51.9% of observation construction.
- Second-largest: fingerprint row extraction at `1380.667 ms`, 32.7% of observation construction.
- Both costs are prefix-dependent scans over already normalized replay data.
- `PriceFacts` itself is latest-row-only and not the bottleneck.
- Signal facts, structure facts, provenance exclusive work, SHA-256 hashing, and model construction are all small.

Recommended next version: `v0.47.0 Observation Prefix Preparation Optimization`.

Scope should focus on safely preserving semantics while avoiding repeated full-prefix DataFrame normalization work inside the builder. A narrower design should prove exact parity for future-data rejection, unsorted input behavior, duplicate timestamp behavior, symbol/provider validation, UTC and float normalization, fingerprint stability, input immutability, and start/end full-frame semantics. Fingerprint row projection is the adjacent second target, but it should not be optimized by deleting fields or weakening canonical semantics.
